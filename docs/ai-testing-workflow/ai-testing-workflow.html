PHASE 1
SRE — Specification Recovery & Static Analysis
Reverse-engineer behavior from code, compare with spec, detect conflicts
STATIC ANALYSIS
  • Parse & Understand Spec
  • Static Code Analysis (per layer)
  • Extract Business Rules
  • Extract Validation Rules
  • Extract Input Domains
  • Infer State / Workflow Rules
  • Map Auth / Authorization Logic
CONFIDENCE SCORE (per item)
HIGH Code/spec clear, 1:1 match, no inference
MED Complex logic, inference required
LOW Ambiguous / no spec / runtime branches unverified
REQUIREMENT COMPARISON & CONFLICT DETECTION
NO CONFLICT → Continue to Phase 2
CONFLICT DETECTED → Classify below
  • Likely Implementation Bug
  • Likely Specification Bug
  • Both Wrong / Ambiguous
  • Need Human Review
External Reference Lookup — used ONLY when conflict is confirmed.
Must be recorded and referenced in AI Audit Report.
OUTPUT
  • Recovered Requirements
  • Input Domains (from code)
  • Business Rules (from code)
  • Validation Rules (from code)
  • State / Workflow Rules
  • Bug Candidates + Confidence Score
  • Requirement Conflicts Report
  • Need External Validation? (Y/N)
